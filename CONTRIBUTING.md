# Contributing to SmartMeetPoint

Thanks for your interest in contributing! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/<your-username>/SmartMeetPoint.git`
3. Create a branch: `git checkout -b feature/your-feature`
4. Follow the [setup instructions](README.md#getting-started) to run the project locally

## Making Changes

- Keep commits focused and atomic
- Write clear commit messages (e.g. `fix: resolve map marker overlap`)
- Test your changes locally before submitting

## Pull Requests

1. Push your branch to your fork
2. Open a Pull Request against `main`
3. Describe what your PR does and link any related issues
4. Wait for review

## Reporting Issues

- Use the GitHub Issues tab
- Include steps to reproduce, expected behavior, and screenshots if applicable

## Code Style

- Use consistent indentation (2 spaces)
- Follow existing patterns in the codebase
- Add comments for non-obvious logic
